# Результаты тестирования: iOS TMA (Buy / Invite)

Дата фиксации: 2026-04-20

## Контекст

Проверялось поведение Telegram Mini App на iPhone при действиях:

- `Buy Silver / Buy Gold`
- `Пригласить друга (+бонус)`

Цель: после нажатия пользователь должен сразу увидеть чат/результат действия.

## Наблюдения до правки

- На iPhone Mini App оставался открыт поверх чата.
- Сообщение в чате отправлялось, но визуально пользователь его не видел сразу.
- На desktop Telegram поведение выглядело лучше: окно обычно скрывалось автоматически.

## Что было добавлено в код

- После `openTelegramLink(...)` добавлен вызов `Telegram.WebApp.close()` с короткой задержкой.
- Применено в:
  - `public/app/seeker-jobs.html` (Buy + Invite)
  - `public/app/pricing.html` (Buy)

## Факт по API Telegram

- `Telegram.WebApp.close()` закрывает Mini App.
- Отдельного программного метода "свернуть, но оставить в стеке для возврата" в WebApp API нет.

## Текущее решение

- Оставляем текущую реализацию с `close()` для iPhone UX (чтобы сразу был виден чат/результат).
- Дополнительных изменений сейчас не вносим.

## Дополнительно проверено по рефералке

При сценарии `User A` приглашает `User B`:

- если `User B` зашел по реферальной ссылке и связь новая, `User A` получает:
  - бонусные открытия (по `ReferralBonusOpens`),
  - уведомление в чат о начислении бонуса;
- при повторном `/start` тем же `User B` повторного начисления нет (идемпотентно).
